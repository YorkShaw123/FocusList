# 专注清单 FocusList —— 部署与运行文档

本文档面向在自己电脑上首次运行本项目的同学，按步骤操作即可把工程跑起来，并演示各项功能。

---

## 一、环境要求

| 项目 | 要求 | 说明 |
| --- | --- | --- |
| 操作系统 | Windows 10/11、macOS 或 Linux | 任意其一 |
| Android Studio | **Koala (2024.1) 或更高版本** | 自带 Gradle、SDK 管理器 |
| JDK | **JDK 17** | Android Studio 一般已内置，无需单独安装 |
| Android SDK | **Android 14 / API 34**（compileSdk=34） | 在 SDK Manager 中勾选安装 |
| 最低运行系统 | Android 8.0 / API 26 及以上 | minSdk=26 |
| 网络 | 可选 | “每日一言”需联网；离线时自动用内置句子兜底，不影响其它功能 |

> 本项目使用 Kotlin + Gradle（Groovy DSL），版本：AGP 8.5.2 / Gradle 8.7 / Kotlin 1.9.24。

---

## 二、导入并运行（推荐方式：Android Studio）

1. **解压**：将 `FocusList.zip` 解压到一个**全英文、无空格**的路径下，例如 `D:\AndroidProjects\FocusList`。
2. **打开工程**：启动 Android Studio → `File ▸ Open…` → 选择解压出来的 **FocusList 文件夹**（包含 `settings.gradle` 的那一层）→ OK。
3. **等待 Gradle Sync**：首次打开会自动下载 Gradle 8.7 与各依赖库（需几分钟，取决于网速）。底部状态栏出现 “Gradle sync finished” 即表示成功。
   - 如提示缺少 SDK，点击提示中的 “Install missing SDK package(s)” 自动补齐。
4. **准备一台设备**（二选一）：
   - **模拟器**：`Device Manager ▸ Create Device` → 选 Pixel 系列 → 系统镜像选 **API 26 及以上**（建议 API 34）→ Finish，然后启动它。
   - **真机**：手机打开 “开发者选项 → USB 调试”，用数据线连接电脑并授权。
5. **运行**：点击工具栏绿色 ▶ `Run 'app'`，选择目标设备。安装完成后 App 会自动启动。

## 三、命令行运行（可选）

工程已附带 Gradle Wrapper，命令行下也可构建：

```bash
# 进入工程根目录
cd FocusList

# Windows
gradlew.bat assembleDebug

# macOS / Linux
./gradlew assembleDebug
```

生成的安装包位于 `app/build/outputs/apk/debug/app-debug.apk`，可用 `adb install app/build/outputs/apk/debug/app-debug.apk` 安装到设备。

## 四、首次启动须知

- App 第一次启动会**申请通知权限**（Android 13+），请点击“允许”，否则任务提醒与番茄钟常驻通知无法显示。
- 数据全部保存在设备本地（Room 数据库 + SharedPreferences），卸载 App 即清除。

## 五、功能演示路线（建议按此顺序验收）

### 1. 待办管理（Activity / 基础控件 / 适配器视图 / 数据持久化）
- 点击右下角 **+** 进入“添加任务”页：输入标题、选择分类（Spinner）、优先级（Chip）。
- 打开“设置到期提醒”开关，会弹出**日期 + 时间选择器**，选择 1~2 分钟后的时间用于测试提醒。
- 保存后返回列表，可见任务卡片（左侧色条表示优先级）。
- **勾选**复选框标记完成（标题出现删除线）；**左右滑动**任意任务可删除，并弹出“撤销”。
- 点击任务卡片可再次进入编辑。

### 2. 任务到点提醒（BroadcastReceiver + AlarmManager + 通知）
- 用上一步设置了提醒的任务，等待到设定时间（可把任务设为 1 分钟后）。
- 到点后系统通过 `AlarmManager` 触发 `ReminderReceiver`，弹出一条**任务提醒通知**。

### 3. 番茄钟专注（Service 前台服务 + 多媒体）
- 底部切到 **专注** 页：拖动滑块选择时长，下拉框可关联一个待办任务，点击 **开始专注**。
- 启动后倒计时圆环开始走动；**退到桌面或锁屏，计时仍在前台服务中持续**，通知栏有常驻通知（带暂停/继续/结束按钮）。
- 倒计时结束会**播放提示音并振动**（多媒体 API）。若在“设置”里开启“白噪音”，专注期间会循环播放环境音。
- 也可在 **待办列表**点击某条任务上的“专注”按钮，一键对该任务开始专注。

### 4. 数据统计（LiveData 响应式刷新）
- 底部切到 **统计** 页：查看今日/累计专注分钟数、番茄个数、待办数量，以及最近专注记录列表。完成一次专注后返回此页，数据会自动更新。

### 5. 内容提供器（ContentProvider）
- 在 **统计** 页点击 **“通过内容提供器查询任务”** 按钮：App 通过 `ContentResolver` 调用 `TaskProvider` 统计任务数量并弹出结果，直观演示内容提供器对外提供数据。

### 6. 桌面小组件（ContentProvider 跨组件协作）
- 回到手机桌面，长按空白处 → **小组件 / Widgets** → 找到 **专注清单** → 拖到桌面。
- 小组件展示“待办 / 已完成”数量，这些数字正是通过 `ContentProvider` 读取的；点击小组件上的刷新按钮可重新读取，点击主体可打开 App。

### 7. 网络请求（Retrofit）
- 专注页顶部绿色卡片是“每日一言”，由 Retrofit 请求 `https://v1.hitokoto.cn/` 获取；点击右上角刷新图标可换一句。断网时显示内置句子。

### 8. 设置（SharedPreferences 持久化 + 多媒体测试）
- 右上角菜单进入 **设置**：调整默认专注时长、开关提示音/白噪音/振动，并可点击“试听 / 试振”立即体验。设置项重启 App 后依然保留。

## 六、常见问题

**Q1：Gradle Sync 卡住或失败？**
检查网络；首次需要下载依赖。可在 `File ▸ Settings ▸ Build ▸ Gradle` 确认使用项目自带的 Gradle 8.7。必要时点击 `Sync Project with Gradle Files`（大象图标）重试。

**Q2：提示 “SDK location not found”？**
在工程根目录新建 `local.properties`，写入你的 SDK 路径，例如：
`sdk.dir=C\:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk`（macOS/Linux 形如 `sdk.dir=/Users/xxx/Library/Android/sdk`）。Android Studio 打开工程时通常会自动生成该文件。

**Q3：收不到提醒通知 / 番茄钟没有常驻通知？**
确认首次启动时已**允许通知权限**；可在 系统设置 → 应用 → 专注清单 → 通知 中手动开启。

**Q4：每日一言一直是同一句？**
可能处于离线状态或接口限流，属正常兜底；联网后点击刷新即可获取新句子。

**Q5：模拟器没有“小组件”入口？**
长按桌面空白处选择 Widgets；部分模拟器需在桌面（Launcher）中操作，确保已退出 App 回到桌面。

        │   ├── receiver/    # 广播接收器（提醒 / 开机）

        │   ├── ui/          # 各 Activity / Fragment / ViewModel / Adapter
        │   └── util/        # 通知 / 闹钟 / 音效 / 时间 / 偏好设置
