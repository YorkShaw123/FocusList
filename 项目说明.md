# 专注清单 FocusList —— 项目说明

> 移动应用系统综合实践 · 北京信息科技大学
> 一款「待办清单 + 番茄钟专注」Android 应用，使用 Kotlin 实现，完整覆盖 Android 四大组件与课程要求的全部知识点。

---

## 一、项目简介

**专注清单 FocusList** 把“任务管理”和“番茄工作法”结合在一起：用户先把要做的事记成待办，再针对某个任务开启番茄钟专注计时；专注结束后自动统计时长，帮助用户量化和坚持自己的专注习惯。

选择这一选题，是因为它能让 Android **四大组件自然且必要地协同工作**，而不是为了凑组件而生硬堆砌：
- 番茄钟必须在后台持续计时 → **前台服务**；
- 任务需要到点提醒、设备重启后提醒不丢 → **广播接收器**；
- 数据要能被桌面小组件读取 → **内容提供器**；
- 多页面交互 → **多 Activity**。

## 三、整体架构

```
┌──────────────────────────────────────────────────────────┐
│                         UI 层                             │
│  MainActivity(BottomNav) ─ TaskList / Focus / Stats 三大 Fragment │
│  AddEditTaskActivity   SettingsActivity                   │
│            │ 观察 LiveData / 调用                          │
│        ViewModel(Task / Focus / Stats)                    │
└───────────────────────────┬──────────────────────────────┘
                            │
                    ┌───────▼────────┐
                    │   Repository    │  单一数据源
                    └───┬────────┬────┘
            ┌───────────▼──┐  ┌──▼─────────────┐
            │ Room(DAO/DB) │  │ Retrofit(网络) │
            └──────┬───────┘  └────────────────┘
                  │ 同一个 SQLite
        ┌──────────▼───────────┐
        │   ContentProvider     │◀── 桌面小组件 / 统计页 通过 ContentResolver 读取
        └──────────────────────┘

  Service(前台番茄钟) ── 计时/多媒体 ──▶ 通知；状态用 LiveData 回传 UI
  BroadcastReceiver(提醒/开机) ── AlarmManager 触发 ──▶ 通知
```

## 四、Android 四大组件落地（重点）

| 四大组件 | 本项目实现 | 文件 | 作用 |
| --- | --- | --- | --- |
| **Activity 活动** | MainActivity、AddEditTaskActivity、SettingsActivity（+ 三个 Fragment） | `ui/**` | 承载待办、专注、统计、增改、设置等界面与交互 |
| **Service 服务** | FocusTimerService（**前台服务**，`foregroundServiceType=specialUse`） | `service/FocusTimerService.kt` | 番茄钟倒计时在后台持续运行，并循环播放白噪音 |
| **BroadcastReceiver 广播接收器** | ReminderReceiver（任务提醒）、BootReceiver（开机重排） | `receiver/**` | 接收 AlarmManager 定时广播弹出提醒；接收 `BOOT_COMPLETED` 重新排程 |
| **ContentProvider 内容提供器** | TaskProvider（`content://com.bistu.focuslist.provider/tasks`） | `provider/TaskProvider.kt` | 以标准 Uri 对外（含跨进程的桌面小组件）暴露任务数据 |

> 组件协作示例：在待办列表点“专注”→ 启动 **Service** 计时；设置提醒 → **BroadcastReceiver** 到点通知；桌面 **小组件** 通过 **ContentProvider** 读取任务数 → 点击打开 **Activity**。

## 六、数据模型

**tasks（任务表）**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Long(主键自增) | 任务 id |
| title | String | 标题 |
| notes | String | 备注 |
| category | String | 分类（学习/工作/生活/其他） |

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | Long(主键自增) | 记录 id |
| taskId | Long? | 关联任务（可空，自由专注） |
| taskTitle | String | 任务标题快照 |
| durationMinutes | Int | 专注时长（分钟） |
| startTime / endTime | Long | 起止时间戳 |
| completed | Boolean | 是否完整完成 |

## 七、主要模块说明

- **data/**：`Task`、`FocusSession` 实体；`TaskDao`、`FocusSessionDao`（查询返回 LiveData，写操作为挂起函数）；`AppDatabase` 单例；`Repository` 统一数据入口。
- **network/**：`QuoteApi` 接口、`QuoteResponse` 模型、`RetrofitClient` 单例。
- **provider/**：`TaskProvider` 复用 Room 的 SQLite，支持 query/insert/update/delete 与变化通知。
- **service/**：`FocusTimerService` 前台计时、记录会话、多媒体；`FocusUiState` 通过 LiveData 把状态回传界面。
- **receiver/**：`ReminderReceiver` 弹出提醒；`BootReceiver` 开机后 `goAsync()` 重排闹钟。
- **widget/**：`TaskWidgetProvider` 通过 `ContentResolver` 查询任务数量并刷新桌面小组件。
- **ui/**：`MainActivity` + 三个 Fragment（待办/专注/统计）+ 各自 ViewModel/Adapter，`AddEditTaskActivity`、`SettingsActivity`。
- **util/**：`NotificationHelper`（通知渠道）、`AlarmScheduler`（AlarmManager）、`SoundManager`（多媒体）、`TimeUtils`、`Prefs`。
